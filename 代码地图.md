# ERP14 前端代码地图

本文档用于快速定位当前可编辑源码、后端接口和仍在使用的生产单文件。优先搜索函数名或 DOM id；行号只表示本文档最近复核时的位置，代码变化后以 `rg -n` 的结果为准。

## 使用顺序

1. 功能修改优先定位 `frontend/index.html`、`frontend/css/main.css`、`frontend/js/main.js`。
2. 后端接口定位 `backend/server.js`；数据库结构定位 `backend/db/schema.sql`。
3. 使用 `scripts/build-frontend-preview.js` 生成预览，不直接覆盖生产页。
4. C 盘完成验收后，只有获得明确授权并完成 O 盘同步准备，才能从 O 盘部署 `erp14-server-showcase.html`。
5. 已撤回或已完成的规格与计划统一保存在 `docs/archive/`，不得把废弃计划当作当前需求。

## 生产页预检脚本

**文件**：`scripts/build-frontend-preview.js`

将 `frontend/` 拆分源码合并为单页预览 HTML，用于检查拆分后的代码合并后是否正常。

**功能**：
- 读取 `frontend/index.html`（HTML 结构）
- 读取 `frontend/css/main.css`（内嵌到 `<style>`）
- 读取 `frontend/js/main.js`（内嵌到 `<script>`）
- 输出到 `dist/erp14-server-showcase.preview.html`

**注意**：
- 此文件**仅供预检**，不是正式部署文件。
- **正式生产文件仍是 `erp14-server-showcase.html`**。
- 不覆盖 `erp14-server-showcase.html`。
- 不修改 `frontend/` 下的任何文件。

**使用方法**：
```bash
node scripts/build-frontend-preview.js
```

## 一键验证总脚本

**文件**：`scripts/verify-all.js`

## 修改代码的建议流程

为了保证 `erp14-server-showcase.html` 与 `frontend/` 保持一致，建议按以下顺序操作：

1. **改主源**：先改 `erp14-server-showcase.html`（所有功能修改以此文件为准）。
2. **同步 frontend/**：将对应改动同步到 `frontend/`：
   - CSS 改 → 同步到 `frontend/css/main.css`
   - HTML 结构改 → 同步到 `frontend/index.html`
   - JS 逻辑改 → 同步到 `frontend/js/main.js`
3. **跑同步检查**：`node scripts/check-frontend-sync.js`，确认无报告差异。
4. **跑验证**：`node scripts/verify-all.js`，确认全部验证通过。
5. **记录日志**：在 `更新日志.md` 中记录本次修改。
