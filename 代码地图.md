# ERP14 前端代码地图

本文档用于快速定位 `erp14-server-showcase.html` 和拆分后的 `frontend/` 文件。

## 当前文件关系

| 文件 | 作用 | 说明 |
| --- | --- | --- |
| `erp14-server-showcase.html` | **主生产文件**（单文件） | 内嵌全部 CSS + HTML 主体 + JS。所有功能修改首先以本文件为准。 |
| `frontend/index.html` | 从主 HTML 拆分出的页面结构源码镜像 | 引用 `./css/main.css` 和 `./js/main.js`。 |
| `frontend/css/main.css` | 从 `<style>` 提取出的样式源码镜像 | 保持同步。 |
| `frontend/js/main.js` | 从 `<script>` 提取出的前端逻辑源码镜像 | 保持同步。 |
| `backend/server.js` | 后端 API 服务 | 负责登录、配置、活动、建议、图片、导入导出等接口。 |

> **重要**：生产实际使用的文件仍是 `erp14-server-showcase.html`（单文件），`frontend/` 仅供拆分开发和代码复用参考。

## 同步检查脚本

**文件**：`scripts/check-frontend-sync.js`

检测 `erp14-server-showcase.html` 与 `frontend/` 拆分源码是否同步。

**检查项**（共约 80 项）：
- 文件存在性
- CSS/JS 行数近似度
- 关键函数在 HTML 和 main.js 中同时存在（约 28 个函数）
- DOM 结构关键元素在 index.html 中存在
- 13 个面板入口在 HTML 和 index.html 中都存在
- 主要 CSS 类名一致性
- 常用 DOM id 一致性

**使用方法**：
```bash
node scripts/check-frontend-sync.js
```

输出示例：
```

  ...
=== 检查结果 ===
```

## 生产页预检脚本

**文件**：`scripts/build-frontend-preview.js`

将 `frontend/` 拆分源码合并为单页预览 HTML，用于检查拆分后的代码合并后是否正常。

**功能**：
- 读取 `frontend/index.html`（HTML 结构）
- 读取 `frontend/css/main.css`（内嵌到 `<style>`）
- 读取 `frontend/js/main.js`（内嵌到 `<script>`）
- 输出到 `dist/erp14-server-showcase.preview.html`

**注意**：
- 此文件**仅供预检**，不是正式部署文件。
- **正式生产文件仍是 `erp14-server-showcase.html`**。
- 不覆盖 `erp14-server-showcase.html`。
- 不修改 `frontend/` 下的任何文件。

**使用方法**：
```bash
node scripts/build-frontend-preview.js
```

## 一键验证总脚本

**文件**：`scripts/verify-all.js`

统一验证入口，一个小步任务完成后优先运行这个总脚本，而不是手工分别运行多条命令。

**运行方式**：
```bash
node scripts/verify-all.js
```

**执行顺序**：
1. 同步检查（`check-frontend-sync.js`）
2. 生产页预检（`build-frontend-preview.js`）
3. preview 构建测试（`test-build-frontend-preview.js`）
4. HTML 内联 JS 语法检查
5. `frontend/js/main.js` 语法检查
6. 后端测试（`backend` 目录 `npm test`）
7. 全部 `scripts/test-*.js` 专项测试（跳过第 3 步已运行的）

**注意**：
- 任意步骤失败时立即停止，不会假通过。
- 不修改业务代码，不覆盖生产页，不部署。

## 修改代码的建议流程

为了保证 `erp14-server-showcase.html` 与 `frontend/` 保持一致，建议按以下顺序操作：

1. **改主源**：先改 `erp14-server-showcase.html`（所有功能修改以此文件为准）。
2. **同步 frontend/**：将对应改动同步到 `frontend/`：
   - CSS 改 → 同步到 `frontend/css/main.css`
   - HTML 结构改 → 同步到 `frontend/index.html`
   - JS 逻辑改 → 同步到 `frontend/js/main.js`
3. **跑同步检查**：`node scripts/check-frontend-sync.js`，确认无报告差异。
4. **跑验证**：`node scripts/verify-all.js`，确认全部验证通过。
5. **记录日志**：在 `更新日志.md` 中记录本次修改。
