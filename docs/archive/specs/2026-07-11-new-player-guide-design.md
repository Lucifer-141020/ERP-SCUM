# 新手入服引导卡设计

> **状态：已废弃 / 已被头图快捷操作替代**
>
> 本规格中的“五步加入服务器”整块引导已经撤回。当前实现是在首页头图信息区点击“IP 地址”复制服务器 IP，点击“QQ群”打开加群页面；不得依据本文恢复步骤 3、4、5 或整块引导卡。

## 目标

在首页主视觉与首页数据卡之间增加“5 步加入服务器”引导，让第一次访问网站的玩家可以按顺序完成入服准备。

成功标准：玩家无需寻找分散信息，即可复制服务器 IP、复制 QQ 群号、阅读规则、了解如何进入服务器，并知道遇到问题时如何联系管理员。

## 范围

- 只修改前台源码，不改后端、数据库和后台表单。
- 复用现有 `serverInfo.ip` 和 `serverInfo.group`，不增加重复配置。
- 不改服务器主视觉、活动报名、玩家建议等无关功能。
- 生产页 `erp14-server-showcase.html` 暂不切换，继续沿用现有发布边界。

## 页面位置与结构

引导区域放在首页主视觉区域之后、首页数据卡之前，标题为“5 步加入服务器”，副标题为“照着操作，几分钟即可完成入服准备”。

五个步骤按以下顺序展示：

1. 复制服务器 IP。
2. 加入 QQ 群。
3. 阅读规则。
4. 进入服务器。
5. 遇到问题联系管理员。

桌面端采用五列横向步骤；窄屏自动切换为单列纵向步骤，避免文字和按钮拥挤。

## 交互

### 复制服务器 IP

按钮读取当前 `serverInfo.ip`。复制成功后提示“服务器 IP 已复制”。如果现代剪贴板接口不可用，则使用现有兼容复制方式；仍失败时明确提示玩家手动复制页面上的 IP。

### 加入 QQ 群

采用已确认的方案 A：按钮读取当前 `serverInfo.group` 并复制群号。成功后提示“群号已复制，请打开 QQ 搜索并申请加入”。不依赖 QQ 群链接。

### 阅读规则

点击后平滑滚动至现有 `homeRulesSection`。不新增规则副本，避免两处内容不一致。

### 进入服务器

点击后显示简短提示：打开 SCUM，使用已经复制的服务器 IP 搜索或直连。本阶段不尝试调用游戏客户端协议。

### 联系管理员

点击后提示玩家进入 QQ 群联系管理员；不增加新的联系方式字段。

## 数据与依赖

引导卡只读取前端已经加载的 `serverInfo`。当后台配置或本地配置更新 IP、QQ群号后，引导卡与主视觉同步显示最新值。

本功能不发起新接口请求，不保存玩家状态，不写入本地存储。

## 可访问性与移动端

- 所有操作使用原生按钮，并保留键盘焦点。
- 操作结果通过现有提示组件反馈。
- 在 320、375、414 像素宽度检查文字换行、按钮点击区域、卡片溢出和横向滚动。
- 不仅依靠颜色表达步骤含义，每一步都有序号、标题和文字说明。

## 验证

- 检查五个步骤顺序、文字和按钮均正确展示。
- 修改模拟配置后，确认 IP 与 QQ 群号同步更新。
- 验证复制成功提示和复制失败降级提示。
- 验证“阅读规则”能够定位到现有规则区域。
- 验证 320、375、414 像素宽度无横向溢出。
- 运行现有前端语法与首页结构检查，并通过本地 HTTP 预览验证。

## 不在本次范围

- 后台可编辑引导步骤。
- 玩家账号、入服进度保存或完成状态。
- QQ 群链接唤起。
- SCUM 客户端一键启动。
- 活动报名、玩家建议和全站视觉改版。
