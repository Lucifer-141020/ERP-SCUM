## 主文件

- 网页入口：`erp14-server-showcase.html`
- 当前项目是单文件静态网页，主要数据暂时保存在浏览器 `localStorage`，也支持后续接真实后端配置。

## CSS 分层

样式在同一个 `<style>` 里，当前按维护目的分为：

1. 基础变量和通用控件：颜色变量、按钮、输入框、卡片。
2. 前台页面：首页主图、玩法、玩家建议、每周活动、建筑模板、服规。
3. 后台管理：后台分组、上传、保存、玩家建议管理。
4. 弹窗和响应式：提交建议弹窗、大图预览、手机导航。
5. 覆盖层：荒野据点主题、暗色模式、后续针对问题做的修复。

注意：有些 CSS 选择器会重复出现，这是为了后面的主题和移动端覆盖前面的基础样式。不要只因为选择器重复就删除，必须先确认两段内容完全相同。

## 后台常用入口

- 首页大标题：管理 -> 网站编辑 -> 首页大标题
- 首页主图轮播：管理 -> 网站编辑 -> 轮播图地址 / 上传首页轮播图
- 玩家群：管理 -> 玩家群设置
- 建筑模板：管理 -> 建筑模板
- 玩家建议：管理 -> 玩家建议
- 图片库：管理 -> 图片库
- 备份导入导出：管理 -> 备份恢复

## 当前玩家建议规则

- 前台只显示：待讨论、已完成、已拒绝。
- 已计划数据在前台归入待讨论，后台暂时保留，避免旧数据丢失。
- 只有待讨论建议可以投票。
- 已完成和已拒绝只展示投票数字、处理说明或拒绝原因。

正式环境密钥只通过服务器环境变量提供：`ADMIN_PASSWORD` 和 `JWT_SECRET`。不得把真实值写入代码、日志、GitHub 或文档。

## 每次修改后的验证

建议至少检查：

1. 页面访问返回 200。
2. 抽取 `<script>` 后执行 `node --check`。
3. 浏览器检查首页、玩家建议、管理页是否有控制台错误。
4. 手机宽度检查是否横向溢出。
5. 如果改了上传、建议、后台保存，必须实测一次对应交互。

## 清理原则

- 可以删除完全相同的重复 CSS。
- 不要删除主题覆盖、暗色模式覆盖、移动端覆盖。
- 不要删除 `panelSaveState` 的多个模板引用；它们在不同后台面板中分时渲染，不是同时存在的重复 DOM。
- 临时脚本执行后应删除，避免后期误运行覆盖页面。
