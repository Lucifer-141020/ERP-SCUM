# ERP14 回滚说明

## 触发条件

出现启动失败、后台无法登录、保存异常、图片异常、页面结构异常或严重控制台错误时，停止继续发布并准备回滚。

## 回滚顺序

1. 停止新版本后端服务。
2. 恢复部署前生产网页备份。
3. 恢复原后端代码。
4. 保留正式数据库和上传图片，除非明确确认需要恢复数据备份。
5. 恢复原环境变量和服务配置。
6. 重启后端服务并检查首页、登录和数据读取。

## 注意事项

- 不用 C 盘测试数据库覆盖正式数据库。
- 不用代码发布包覆盖 `backend/data/` 或 `backend/uploads/`。
- 回滚后记录时间、原因、恢复文件和验证结果。
- 真实密码、JWT 密钥和数据库内容不得写入文档或日志。
