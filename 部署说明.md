# ERP14 部署说明

## 部署边界

- C 盘只负责开发、测试和验收。
- O 盘是正式同步、归档和部署来源。
- 未经总裁明确确认，不上传服务器、不重启 PM2、不覆盖生产页。

## 部署前提

1. C 盘全量验收通过。
2. O 盘完成同步并重新检查 Git 状态。
3. 服务器数据库、上传图片、生产网页和环境配置完成备份。
4. 发布包不包含 `backend/data/`、`backend/uploads/`、`.env`、备份和依赖缓存。

## 发布包

使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\deploy\build-code-release-package.ps1
```

发布包应包含前端单文件和后端运行所需源码、依赖锁定文件、配置模块、中间件和数据库结构文件。

## 生产环境变量

只在服务器设置，不写入代码或 Git：

```text
NODE_ENV=production
PORT=3000
ADMIN_PASSWORD=<真实管理员密码>
JWT_SECRET=<真实随机密钥>
ERP14_DB_PATH=<服务器数据库路径>
```

## 上线顺序

1. 上传到临时目录。
2. 安装依赖并执行启动检查。
3. 使用临时端口验证登录、保存和读取。
4. 备份当前生产文件。
5. 切换正式代码并重启服务。
6. 验收首页、后台、图片、活动、主题、移动端和控制台。

正式部署完成前，不得声称已经上线。
