## 二、当前有效功能

### 1. 首页头图快捷操作

- “服务器编号”普通展示。
- 点击“IP 地址”整块复制服务器 IP。
- 点击“QQ群”整块打开加群页面。
- 手机端点击区域不少于44px。
- 原“五步加入服务器”整块引导已经撤回，不得恢复步骤3、4、5。

### 2. 活动报名前台体验

- 支持“常驻、报名中、即将截止、报名已截止、进行中、已结束、已颁奖”等状态。
- 状态与倒计时共用统一时间判断。
- 报名成功后使用 `erp14-event-signups` 保存本设备记录，按钮显示“玩家已报名”。
- 记录只用于本设备体验，不代表账号级报名凭证；重复报名仍由后端拦截。
- 已截止、已结束、已颁奖时不显示报名按钮。
- 已颁奖活动不会自动消失，由管理员手动取消发布或删除。

### 3. 移动端触控

- 320/375/414 竖屏下菜单和主题按钮为44×44px。
- `max-height:450px` 且 `pointer:coarse` 的触控横屏同样为44×44px。
- 非触控桌面继续保持38×38px基础规格。
- 手机及触控横屏的高频 CTA 和后台通知保存按钮具备44px触控高度。

### 4. 玩家建议卡片

- 前台直接展示建议正文、图片和管理员说明。
- 已撤回“查看详情/收起详情”卡片内展开方案。
- 底部只保留“同意、否定、讨论”。
- 管理员说明按状态显示“管理员回复、完成说明、拒绝原因”；空字段不渲染说明节点。
- 卡片正文、按钮和管理员说明完成字体与换行优化。

### 5. 玩家建议图片闭环

- 前端提交使用 `content` 和 `images`，等待后端成功后才写入本地列表。
- 失败时保留弹窗和待上传图片，并显示后端真实错误。
- `normalizeRequestItem()` 统一处理 `content/text`、管理员字段和图片 JSON。
- 公共 `GET /api/requests` 返回 `contact`、`images`、`reject_reason`，图片解析失败安全回退 `[]`。
- 前台和后台缩略图复用现有 `imageViewer` 查看大图。
- 后台缩略图包含明确的“查看大图”提示和可访问名称。

### 6. 服务器通知后台编辑

- 后台“首页内容”面板可编辑启用状态、标题和逐行通知内容。
- 复用现有 `PUT /api/admin/config`，配置键为 `server_notice`；未新增专用 API 或数据库表。
- 标题和通知行使用 `textContent` 安全渲染，不执行管理员输入的 HTML 或脚本。
- 使用 `erp14-notice-dismissed-version` 保存关闭版本；新版本自动重新显示。
- 保存失败保留表单、内存配置和旧版本，并显示真实错误。
- 保存期间阻止重复请求。
- 手机端通知保持右下角浮动，关闭按钮可见，后台保存按钮达到44px。
