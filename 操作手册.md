# ERP14 SCUM 用户网页操作手册

本手册用于记录 ERP14 SCUM 用户网页从本地开发、GitHub 版本管理、服务器上线到后期维护更新的完整操作流程。

适用项目目录：

```text
O:\AI\Open AI_Codex\SCUM用户网页
```

GitHub 仓库：

```text
https://github.com/Lucifer-141020/ERP-SUCM
```

## 1. 项目文件说明

### 1.1 核心文件

```text
erp14-server-showcase.html
```

这是前台网页和后台管理页面的主文件。网页样式、页面结构和大部分前端交互都在这个文件里。

```text
backend/server.js
```

这是后端服务。后端负责读取配置、保存后台修改、处理图片上传、提供接口。

```text
backend/data/site-config.json
```

这是当前网站的运行数据。后台保存后的配置会写到这里。

注意：这个目录不提交到 GitHub，正式服务器上要单独保存和备份。

```text
backend/uploads
```

这是后台上传图片的目录。

注意：这个目录不提交到 GitHub，正式服务器上要单独保存和备份。

```text
erp14-site-config.example.json
```

这是配置示例文件，只放示例数据，可以提交到 GitHub。

```text
README.md
ERP14-维护说明.md
操作手册.md
```

这些是说明文档。

## 2. 前期网页注意事项

### 2.1 不要把真实密码写进代码

真实后台密码、管理员令牌、数据库密码、服务器密码都不能写进以下文件：

```text
erp14-server-showcase.html
backend/server.js
README.md
操作手册.md
```

示例只能写成：

```text
YOUR_ADMIN_PASSWORD_HERE
YOUR_RANDOM_TOKEN_HERE
```

### 2.2 GitHub 只存代码，不当数据库

GitHub 用来保存代码版本，不用来保存正式运营数据。

可以提交：

```text
erp14-server-showcase.html
backend/server.js
backend/package.json
backend/test
README.md
ERP14-维护说明.md
操作手册.md
erp14-site-config.example.json
```

不要提交：

```text
backend/data/
backend/uploads/
.env
备份/
*.backup-*.html
*.before-restore-*.html
```

### 2.3 本地和公网接口地址规则

当前网页已经做了自动判断：

本地访问时：

```text
http://127.0.0.1:52395/erp14-server-showcase.html
```

网页会自动连接：

```text
http://127.0.0.1:3000
```

公网域名访问时，例如：

```text
https://example.com
```

网页会自动连接当前域名：

```text
https://example.com
```

这样上线后，Nginx 只需要把 `/api/` 和 `/uploads/` 转发到后端即可。

### 2.4 CSS 修改注意事项

网页是单文件结构，CSS 都在 `erp14-server-showcase.html` 的 `<style>` 中。

有些 CSS 选择器会重复出现，这是为了：

```text
主题覆盖
暗色模式覆盖
移动端覆盖
后续修复覆盖
```

不要只因为看见重复就删除。删除前必须确认两段 CSS 内容完全相同，而且不会影响手机端和暗色模式。

## 5. 上线前准备

### 5.1 服务器选择

当前项目推荐：

```text
轻量应用服务器
2核2G
3Mbps 带宽
40GB 系统盘
Ubuntu 22.04 或 宝塔 Linux 面板
```

项目目前不需要高配置服务器。后期访问量变大、图片变多，再升级即可。

### 5.2 域名准备

如果有域名，把域名解析到服务器公网 IP。

常见解析：

```text
A 记录
主机记录：@ 或 www
记录值：服务器公网 IP
```

### 5.3 安全组和端口

服务器需要开放：

```text
80   HTTP 访问
443  HTTPS 访问
22   SSH 登录
```

后端端口：

```text
3000
```

建议只让服务器本机访问，不直接暴露给公网。公网通过 Nginx 访问 `/api/` 和 `/uploads/`。

## 6. 正式密码和管理员权限

### 6.1 什么时候设置正式密码

正式密码在服务器部署时设置，不在本地代码里设置。

启动后端时通过环境变量设置：

```bash
ADMIN_PASSWORD="YOUR_REAL_ADMIN_PASSWORD"
ADMIN_TOKEN="YOUR_RANDOM_LONG_TOKEN"
PORT=3000
node server.js
```

说明：

```text
ADMIN_PASSWORD：后台登录密码
ADMIN_TOKEN：管理员接口令牌，给程序识别登录状态用
PORT：后端运行端口
```

### 6.2 密码安全要求

后台密码建议：

```text
不少于 12 位
包含大小写字母、数字、符号
不要用生日、QQ、手机号、服务器名
不要发到群里
不要写进 GitHub
```

### 6.3 管理员权限现状

当前第一版逻辑是：

```text
知道后台密码的人就是管理员
```

第一阶段上线够用。

后期如果需要，可以再升级成：

```text
多个管理员账号
不同管理员权限
操作日志
修改记录
```

这些属于第二阶段功能，不建议第一版上线时一起做，避免复杂度过高。

## 7. 服务器上线步骤

### 7.1 拉取 GitHub 代码

服务器进入网站目录，例如：

```bash
cd /www/wwwroot
```

拉代码：

```bash
git clone https://github.com/Lucifer-141020/ERP-SUCM.git
```

进入项目：

```bash

```

### 7.2 上传运行数据

本地这些目录没有提交到 GitHub，需要单独上传到服务器：

```text
O:\AI\Open AI_Codex\SCUM用户网页\backend\data
O:\AI\Open AI_Codex\SCUM用户网页\backend\uploads
```

服务器目标位置：

```text
/www/wwwroot/ERP-SUCM/backend/data
/www/wwwroot/ERP-SUCM/backend/uploads
```

上传完成后，确认存在：

```bash
ls backend/data
ls backend/uploads
```

### 7.3 启动后端

进入后端：

```bash
cd /www/wwwroot/ERP-SUCM/backend
```

启动：

```bash
ADMIN_PASSWORD="YOUR_REAL_ADMIN_PASSWORD" ADMIN_TOKEN="YOUR_RANDOM_LONG_TOKEN" PORT=3000 node server.js
```

正式环境建议用进程守护工具，例如：

```text
PM2
宝塔 Node 项目管理
1Panel 应用管理
systemd
```

进程守护工具的作用是：服务器重启或程序异常退出后，自动拉起后端。

### 7.4 配置 Nginx

Nginx 的作用是：让外网访问域名时，正确打开网页，并把接口请求转给后端。

参考配置：

```nginx
server {
  listen 80;
  server_name 你的域名.com www.你的域名.com;

  root /www/wwwroot/ERP-SUCM;
  index erp14-server-showcase.html;

  location / {
    try_files $uri $uri/ /erp14-server-showcase.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  location /uploads/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

### 7.5 配置 HTTPS

上线后建议开启 HTTPS。

HTTPS 的作用是：让网址显示安全锁，保护登录密码和后台操作数据。

如果使用宝塔或 1Panel，可以在面板里申请免费证书。

申请成功后，访问地址应该是：

```text
https://你的域名.com
```

## 11. 常见问题

### 11.1 网站能打开，但后台保存失败

重点检查：

```text
后端是否运行
Nginx 是否正确转发 /api/
后台密码是否正确
服务器 data 目录是否有写入权限
```

### 11.2 图片上传成功但前台不显示

重点检查：

```text
backend/uploads 是否存在
Nginx 是否正确转发 /uploads/
图片文件是否真实存在
浏览器地址是否能直接打开图片
```

### 11.3 GitHub 上没有 data 和 uploads

这是正常的。

原因：

```text
backend/data 是运行数据
backend/uploads 是上传图片
```

它们不适合放 GitHub，应该放服务器并单独备份。

### 11.4 域名访问接口不对

当前网页会自动用当前域名作为后端地址。

所以正式域名下必须保证：

```text
https://你的域名.com/api/site-config
```

能访问到后端。

如果不能，通常是 Nginx 的 `/api/` 转发没配好。
